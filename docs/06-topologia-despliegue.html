Notas de despliegue

  • Perfiles de compose: --profile postgres o --profile mongo levanta el stack de datos correspondiente. Los app containers reciben DB_ENGINE por env.
  • Sharding Mongo: products.category y reservations.restaurant_id como shard key hashed. Replica set 1 primary + 2 secondaries por shard.
  • Redis allkeys-lru: evicción automática cuando llena memoria; ningún key es "irremplazable".
  • ES: single-node en dev (suficiente para Etapa 2). Cluster 3-node en producción.
  • Escalado: docker compose up --scale api=3 --scale search=2. Nginx descubre las réplicas vía DNS de Docker (resolver 127.0.0.11).
