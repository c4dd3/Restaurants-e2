8 Contrato de API

Endpoints agrupados por servicio, con forma exacta de request/response. Este es el contrato que une cmd/auth, cmd/api y cmd/search, y lo que consumen los tests y el frontend.

Convenciones generales

  • Todos los paths van detrás del balanceador (http://localhost). Nginx rutea por prefijo: /auth/*, /api/*, /search/*.
  • Auth: Authorization: Bearer <jwt>. Endpoints marcados JWT requieren cualquier user; ADMIN requiere role = "admin".
  • Ownership desde el token: user_id nunca se acepta del body. Se extrae de los Claims del JWT en el middleware.
  • Content-Type: siempre application/json.
  • Errores: respuesta uniforme { "error": "slot_not_available", "message": "..." }. El código de error es estable para los clientes.
  • IDs: UUID v4 como string.
  • Fechas: ISO 8601 con timezone (2026-05-07T19:00:00-06:00).

auth-service

base: /auth/* · puerto interno :8081
POST /auth/register PUBLIC

Crea una cuenta nueva. Devuelve JWT listo para usar — no requiere login adicional.

Request body

{
  "name":     "María Pérez",
  "email":    "maria@example.com",
  "password": "s3cure-pw",
  "role":     "client"
}

201 Created

{
  "token": "eyJhbGci...",
  "user": {
    "id": "9f1b...",
    "name":  "María Pérez",
    "email": "maria@example.com",
    "role":  "client",
    "created_at": "..."
  }
}

Errores

400 validation_failed · 409 email_already_exists
POST /auth/login PUBLIC

Valida email + password y firma un JWT con HS256.

Request body

{
  "email":    "maria@example.com",
  "password": "s3cure-pw"
}

200 OK

{
  "token": "eyJhbGci...",
  "user":  { ... }
}

Errores

400 validation_failed · 401 invalid_credentials
GET /auth/health PUBLIC

Liveness probe para nginx y docker.

{ "status": "ok", "service": "auth" }

api-service

base: /api/* · puerto interno :8080
GET /api/users/me JWT

Perfil del usuario autenticado (del JWT, no del path). No hay /users/:id público.

200 OK

{
  "id": "9f1b...", "name": "María Pérez",
  "email": "maria@example.com", "role": "client",
  "created_at": "...", "updated_at": "..."
}
PATCH /api/users/me JWT

Actualiza nombre o email del propio usuario. Campos opcionales.

Request body

{ "name": "María P.", "email": "maria.p@example.com" }

Errores

409 email_already_exists
GET /api/restaurants PUBLIC

Lista paginada. Parámetros: ?page=1&size=20&q=texto (q filtra por nombre).

200 OK

{
  "items": [ { "id": "...", "name": "...", ... } ],
  "page":  1, "size": 20, "total": 42
}
GET /api/restaurants/:id PUBLIC

Detalle. Incluye menús (sin productos — los productos se piden por menú).

Errores

404 restaurant_not_found
POST /api/restaurants ADMIN

admin_id se toma del JWT.

Request body

{
  "name":        "La Cocina",
  "address":     "Av. Central 123",
  "phone":       "+506 2222-3333",
  "description": "...",
  "capacity":    60
}

201 Created

{ "id": "...", "name": "La Cocina", ... }
PATCH /api/restaurants/:id ADMIN

Solo el admin_id dueño puede editar.

Errores

403 not_owner · 404 restaurant_not_found
DELETE /api/restaurants/:id ADMIN

Cascadea menús, productos, reservas. Las órdenes se preservan (FK RESTRICT).

Respuesta

204 No Content
GET /api/restaurants/:id/menus PUBLIC

Menús de un restaurante. Incluye productos (embebidos en mongo, join en pg).

POST /api/menus ADMIN

Crea un menú y opcionalmente sus productos en la misma operación.

Request body

{
  "restaurant_id": "9f1b...",
  "name":          "Menú del día",
  "description":   "...",
  "products": [
    {
      "name":      "Casado",
      "description": "Plato típico",
      "category":  "principal",
      "price":     4500,
      "available": true
    }
  ]
}

Errores

403 not_owner · 404 restaurant_not_found
PATCH /api/menus/:id ADMIN

Actualiza nombre/descripción. Para modificar productos, usar los endpoints de /products.

DELETE /api/menus/:id ADMIN
POST /api/menus/:id/products ADMIN

Agrega un producto al menú. Side-effect: indexa el producto en Elasticsearch si available = true.

Request body

{
  "name":        "Pizza Margarita",
  "description": "Tomate, mozzarella, albahaca",
  "category":    "pizza",
  "price":       5800,
  "available":   true
}
PATCH /api/products/:id ADMIN

Side-effect: reindexa en ES (o desindexa si pasa a available=false).

DELETE /api/products/:id ADMIN

Side-effect: borra del índice de ES.

Errores

409 product_has_orders (FK RESTRICT)
GET /api/reservations/me JWT

Reservas del propio usuario. Query: ?status=pending|confirmed|cancelled.

POST /api/reservations JWT

Crea una reserva. Valida: fecha futura, party_size > 0, slot disponible. Ver diagrama 5 para el flujo completo.

Request body

{
  "restaurant_id": "9f1b...",
  "date":          "2026-05-07T19:00:00-06:00",
  "party_size":    4,
  "notes":         "mesa cerca de ventana"
}

201 Created

{
  "id": "...",
  "restaurant_id": "...",
  "user_id":       "...",
  "date":          "...",
  "party_size":    4,
  "status":        "pending",
  "notes":         "..."
}

Errores

400 date_must_be_future · 400 party_size_too_large · 404 restaurant_not_found · 409 slot_not_available
PATCH /api/reservations/:id JWT

Solo estado (cancelled) y notas. No se puede mover fecha — se cancela y se crea otra.

GET /api/orders/me JWT

Órdenes del propio usuario.

POST /api/orders JWT

Crea un pedido. El total se calcula en el servidor a partir de los precios actuales de los productos (snapshot). No se confía en el valor del cliente.

Request body

{
  "restaurant_id":  "9f1b...",
  "reservation_id": null,
  "pickup":         false,
  "items": [
    { "product_id": "a1...", "quantity": 2 },
    { "product_id": "b2...", "quantity": 1 }
  ]
}

201 Created

{
  "id": "...", "user_id": "...",
  "restaurant_id": "...",
  "total": 16100.00, "status": "pending",
  "pickup": false,
  "items": [
    { "product_id": "a1...", "quantity": 2, "price": 5800 },
    { "product_id": "b2...", "quantity": 1, "price":  4500 }
  ]
}

Errores

400 empty_items · 404 product_not_found · 409 product_unavailable

search-service

base: /search/* · puerto interno :8082 · lee de Elasticsearch
GET /search/products PUBLIC

Búsqueda textual sobre name + description. Query: ?q=texto&category=pizza&page=1&size=20.

200 OK

{
  "items": [
    {
      "id": "...", "name": "Pizza Margarita",
      "description": "...", "category": "pizza",
      "price": 5800, "restaurant_id": "...",
      "_score": 4.21
    }
  ],
  "page": 1, "size": 20, "total": 17
}
GET /search/products/category/:categoria PUBLIC

Filtro directo por categoría (más rápido que ?q= porque es term query).

POST /search/reindex ADMIN

Reindex completo desde el motor principal. Úsese para bootstrap o recovery. Operación lenta — corre en background.

202 Accepted

{ "job_id": "...", "status": "running" }

Contrato de errores

Todos los errores siguen el mismo shape. El campo error es el código estable que los clientes pueden comparar; message es texto libre para el usuario.

{
  "error":   "slot_not_available",
  "message": "El horario solicitado ya está reservado"
}
  • Códigos estándar: validation_failed (400), invalid_credentials (401), unauthorized (401), forbidden (403), not_owner (403), *_not_found (404), email_already_exists / slot_not_available / product_unavailable (409), internal_error (500).
  • El mapping dominio → HTTP vive en internal/transport/http/errors.go::domainToHTTP. Los services nunca devuelven códigos HTTP; devuelven errores del paquete domain (ErrConflict, ErrNotFound, etc.).
